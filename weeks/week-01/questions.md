# Ответы на вопросы для самопроверки

## 1. В чем разница между "ресурсом" и "коллекцией" в контексте REST API?

**Коллекция** — это множество однотипных ресурсов(cущностей) (например, `/users` — список всех пользователей).  
**Ресурс** — это конкретный объект внутри коллекции (например, `/users/42` — пользователь с ID 42).  
Коллекция — это "папка", ресурс — "файл" внутри неё.

## 2. Почему для создания объекта используется метод `POST` и код ответа `201`, а не `200`?

- `201 Created` точнее отражает суть операции: ресурс **создан**, а не просто обработан запрос.
- Это следует принципам HTTP — разные коды для разных ситуаций.
- Клиент понимает, что запрос успешен и создан новый объект.

## 3. Что такое идемпотентность? Назовите HTTP-методы, которые являются идемпотентными.

**Идемпотентность** — свойство операции, при котором многократное выполнение даёт тот же результат, что и однократное.

Идемпотентные методы: `GET`, `PUT`, `DELETE`, `HEAD`, `OPTIONS`.  
Неидемпотентные: `POST` (каждый раз создаёт новый ресурс).

## 4. Почему ID объекта обычно генерируется на сервере и возвращается в ответе на `POST`?

- Сервер гарантирует уникальность ID (особенно в распределённых системах).
- Клиент может не знать, какие ID свободны.
- Это часть ответственности сервера — управлять идентификацией ресурсов.

## 5. Что такое "контракт API" и почему важно его соблюдать?

**Контракт API** — это договорённость о том, как клиент и сервер общаются: какие эндпоинты есть, какие данные отправлять и что ожидать в ответ.  
Соблюдать его важно, чтобы клиенты не "ломались" при изменениях на сервере, и чтобы разные части системы работали согласованно.

## 6. Зачем команде разработчиков нужна спецификация, например, OpenAPI (Swagger)?

- Автоматическая генерация документации.
- Автоматическая генерация клиентских SDK.
- Упрощение тестирования (Swagger UI позволяет отправлять запросы прямо из браузера).
- Единый источник правды для фронтенда, бэкенда и тестировщиков.
