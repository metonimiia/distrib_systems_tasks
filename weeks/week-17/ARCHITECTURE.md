# Архитектура проекта events-s12

## Общее описание системы
Система **events-s12** (Сервис событий) построена на микросервисной архитектуре и состоит из двух основных компонентов: **API Gateway** и **Events Service**. Взаимодействие между микросервисами осуществляется по протоколу gRPC, а внешние клиенты взаимодействуют с API Gateway по REST.

## Компоненты системы

### 1. API Gateway
- **Назначение**: Точка входа для всех внешних запросов. Маршрутизирует REST запросы к внутренним микросервисам.
- **Технологии**: Python, FastAPI, Uvicorn.
- **Интерфейс**: REST (HTTP).
- **Роуты**:
  - `GET /api/events` — получение списка всех событий.
  - `POST /api/events` — создание нового события.
- **Связность**: Выступает в роли gRPC-клиента для взаимодействия с `Events Service`.

### 2. Events Service (`events-svc-s12`)
- **Назначение**: Внутренний микросервис, отвечающий за бизнес-логику управления событиями (создание, хранение, получение).
- **Технологии**: Python, gRPC (`grpcio`).
- **Интерфейс**: gRPC (порт `8257`).
- **Модель данных**: Сущность `Event` с полями `id` (string), `name` (string), `location` (string).
- **Хранилище данных**: Для упрощения запуска и демонстрации состояния используется In-Memory хранилище (в памяти процесса сервиса). В реальной Production-системе предполагается замена на PostgreSQL.

## Протоколы взаимодействия
- **Фронтенд/Пользователь <-> API Gateway**: REST (HTTP/JSON). Выбран для обеспечения простоты интеграции внешних клиентов.
- **API Gateway <-> Events Service**: gRPC (HTTP/2, Protobuf). Выбран за высокую производительность, строгую типизацию контрактов (proto-файлы) и эффективность бинарной сериализации при межсервисном общении.

## Инфраструктура и развертывание
- **Docker**: Каждый сервис упакован в Docker-контейнер на базе `python:3.11-slim`.
- **Docker Compose**: Для обеспечения удобства разработки ("всё запускается одной командой") написан `docker-compose.yml`, который поднимает `gateway` (порт `8000`) и `events-svc-s12` (внутренний порт `8257`).
- **Observability**: В сервисах реализовано базовое логирование запуска и обработки запросов.
