# Вопросы для самопроверки

1. В чем главное концептуальное отличие GraphQL от REST?
2. Что такое Schema Definition Language (SDL)?
3. Чем Query отличается от Mutation? Могут ли они выполняться параллельно?
4. Что такое "резолвер" (resolver) и кто его вызывает?
5. Как решается проблема N+1 запросов в GraphQL (dataloader)?
6. Почему в GraphQL сложно реализовать кэширование на уровне HTTP (как в REST)?

# README

1. **Главное отличие GraphQL от REST** в том, что в REST сервер определяет структуру ответа через разные эндпоинты, а в GraphQL клиент сам указывает, какие данные и поля ему нужны через один общий endpoint.

2. **SDL (Schema Definition Language)** — это язык описания схемы GraphQL. С его помощью задаются типы, поля, связи между ними, а также `Query`, `Mutation` и другие элементы схемы.

3. **Query** используется для чтения данных, **Mutation** — для изменения данных.  
   Query-поля могут выполняться параллельно, а Mutation обычно выполняются последовательно, чтобы сохранить корректный порядок изменений.

4. **Resolver** — это функция, которая возвращает данные для конкретного поля в GraphQL. Резолверы вызывает сам GraphQL-сервер во время выполнения запроса.

5. Проблема **N+1** решается с помощью **DataLoader**: он объединяет множество одинаковых запросов в один batch и кэширует результаты в рамках одного запроса.

6. В GraphQL сложно использовать **HTTP-кэширование**, потому что обычно все запросы идут на один endpoint (`/graphql`), а сами данные зависят от тела запроса, а не от URL, как в REST.
