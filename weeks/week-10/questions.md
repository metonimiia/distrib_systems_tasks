# Вопросы для самопроверки

1. Что такое слой в Docker и почему порядок команд в Dockerfile важен для кэширования?
   Слой — это результат отдельной команды (`RUN`, `COPY`, `ADD`) в Dockerfile. Порядок важен, потому что Docker кэширует слои: если менять команды в конце, пересобираются только последние слои.
2. В чем суть Multi-stage build? Какие проблемы он решает?
   Multi-stage build разделяет сборку и запуск: на первом этапе ставим/собираем зависимости, во второй образ копируем только нужный результат. Это уменьшает размер образа и убирает лишние инструменты из runtime.
3. Зачем нужен файл `.dockerignore`?
   Он исключает лишние файлы из контекста сборки (`.venv`, кэши, `.git` и т.д.), чтобы ускорить сборку и не раздувать образ.
4. Чем отличаются инструкции `CMD` и `ENTRYPOINT`?
   `CMD` задает команду по умолчанию (ее легко переопределить при `docker run`). `ENTRYPOINT` задает основной процесс контейнера, к которому обычно передаются аргументы.
5. Почему рекомендуется использовать `COPY` вместо `ADD`?
   `COPY` делает только копирование и ведет себя предсказуемо. `ADD` имеет дополнительное поведение (например, авто-распаковка архивов), поэтому его используют только когда это действительно нужно.
6. Почему приложение в контейнере должно слушать `0.0.0.0`, а не `127.0.0.1`?
   `127.0.0.1` доступен только внутри контейнера. `0.0.0.0` слушает на всех интерфейсах контейнера, поэтому сервис становится доступен с хоста через проброс порта.
