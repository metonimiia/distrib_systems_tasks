# Аудит безопасности messages-s12

## Объект проверки

- Проект: `messages-s12`
- Сервис: `messages-svc-s12`
- Порт: `8153`
- Ресурс: `messages`

## Результаты проверки

| Проверка | Результат | Риск | Рекомендация |
|----------|-----------|------|--------------|
| SQL Injection | Критичных проблем не найдено | Низкий | Использовать ORM или параметризованные запросы |
| Command Injection | Вызовы shell-команд не используются | Низкий | Не передавать пользовательский ввод в shell |
| Broken Access Control | Нужна проверка доступа к чужим `messages` по ID | Средний | Проверять владельца ресурса на сервере |
| Секреты в коде | Секреты в коде не найдены | Низкий | Хранить ключи и пароли в переменных окружения |
| Валидация входных данных | Требуется проверять поля запроса | Средний | Валидировать `topic` и другие поля на сервере |
| Security Misconfiguration | Нужно отключать debug в production | Средний | Хранить режим запуска в переменных окружения |

## Проверки вручную

- SQL-инъекция в строковых параметрах: `"' OR '1'='1"` не должна менять результат запроса.
- Подмена ID ресурса: пользователь не должен получать чужие `messages`.
- Подделка токена: сервер должен отклонять неверный или пустой токен.

## Вывод

Критичных уязвимостей не зафиксировано. Основные рекомендации: добавить строгую проверку авторизации, валидировать входные данные и хранить секреты только вне репозитория.
